Protótipo de Envio de Dados

Documentação Técnica e Análise do Projeto

Visão Geral do Projeto

O Protótipo de Envio de Dados é uma interface de autenticação (tela de login ou cadastro) moderna e minimalista. O objetivo principal do projeto foi aplicar conceitos fundamentais de HTML5 estrutural e CSS3 avançado, focando em layouts responsivos e centralizados com Flexbox, validação nativa de formulários e design focado na experiência do usuário (UX).

Detalhes Técnicos e Funcionalidades

1. Estruturação Semântica (HTML5)

O esqueleto da página foi construído de forma limpa e direta, utilizando as melhores práticas de desenvolvimento:

  • Container Principal (.caixa): Uma div isolada que funciona como o "card" do formulário, agrupando os elementos e destacando-os do fundo da tela.
  • Elementos de Formulário (<form>): Uso das tags corretas para captura de dados.
  • Validação Nativa:
    • O input de e-mail usa type="email", forçando o navegador a validar se o usuário digitou um endereço válido (com @ e .com).
    • O input de senha usa type="password" para mascarar os caracteres digitados por segurança, além dos atributos minlength="6" e maxlength="12", que impedem o envio de senhas fracas ou longas demais diretamente pelo navegador.
    • Ambos os campos possuem o atributo required, tornando o preenchimento obrigatório.

2. Estilização e Design (CSS3)

O visual do projeto segue uma paleta de cores escura (Dark Mode) inspirada em editores de código modernos:

  • Reset Global (*): Uso do box-sizing: border-box com zeramento de margens e paddings, garantindo que o tamanho calculado dos elementos seja exato e sem quebras inesperadas.
  • Efeito de Fundo Dinâmico: O corpo da página (body) utiliza um radial-gradient sutil no canto inferior esquerdo (circle at 20% 80%), criando uma atmosfera moderna com um brilho azulado que quebra a monotonia do fundo escuro #0E131F.
  • Centralização Perfeita com Flexbox: O body foi configurado como um container Flex (display: flex) com justify-content: center, align-items: center e uma altura dinâmica de 100vh. Isso garante que o formulário fique perfeitamente centralizado na tela do usuário, independente do tamanho do monitor.
  • Interatividade e Micro-interações:
    • Foco (:focus): Ao clicar nos campos de texto, a borda muda para um tom avermelhado (#772727f8) e o contorno padrão do navegador é removido, criando uma identidade visual própria.
    • Efeito Hover (:hover): O botão de envio altera sua cor de fundo suavemente com uma animação de transição (transition: background 0.3s ease) e muda o cursor do mouse para o formato de clique (cursor: pointer), indicando claramente que o elemento é clicável.

🚀 Aprendizados Adquiridos

Durante o desenvolvimento deste protótipo, foram consolidados conhecimentos práticos em:

  1. Posicionamento com Flexbox: Entendimento real de como alinhar elementos em eixos verticais e horizontais (flex-direction: column para o formulário e eixos cruzados para o body).
  2. Estilização de Estados: Manipulação de pseudo-classes (:hover e :focus) para melhorar a experiência visual do usuário.
  3. Segurança básica em Front-End: Utilização de atributos HTML restritivos para evitar o envio de formulários em branco ou mal formatados.
